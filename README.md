
# 📊 Marketing Analytics Portfolio Project: Comprehensive Case Study

### 🎯 Project Overview and Objective
What we did first was define the overarching goal of this project, which was to conduct a comprehensive **Marketing Analytics** study to evaluate and improve customer engagement and product performance. The objective was to move beyond basic data collection and instead provide a data-driven narrative that helps stakeholders understand customer sentiment and purchasing behavior. By integrating multiple tools, we aimed to transform raw, disconnected data into a centralized, interactive system for strategic decision-making.

### 1. 💡 Identifying and Solving a Business Problem
What we did was identify the core business requirements and the specific questions the marketing team needed to answer. We established the key performance indicators (KPIs) and defined the project's scope to ensure our analysis would provide actionable insights for stakeholders.

### 2. 🗄️ Cleaning Data with SQL for Insights
Then, we proceeded to connect to the SQL Server database to extract raw records from the `fact_customer_reviews` table. We used SQL queries to clean the dataset, ensuring that we handled missing values, removed duplicates, and standardized the data to create a reliable foundation for our analysis. **What we did specifically was store these cleaned queries as SQL Views within the database to ensure they were reusable and easily accessible.**

### 3. 🐍 Advanced Sentiment Analysis with Python
After that, we used Python and the NLTK library to perform an advanced sentiment analysis on customer feedback. What we did was develop a script using the VADER sentiment analyzer to calculate a "compound score" for each review. We then categorized these scores into buckets—such as "Positive," "Mixed Negative," or "Neutral"—by cross-referencing the text sentiment with the numerical star ratings.

### 4. 📉 Building an Interactive Dashboard in Power BI
Next, we proceeded to import our enriched dataset into Power BI. To ensure consistency and performance, what we did was connect Power BI directly to the **SQL Views** we had previously defined in SQL Server. Furthermore, **we introduced a custom script within Power BI to generate a dedicated Date Table (Calendar Table)**, which allowed us to perform advanced time-intelligence calculations. We then created a data model that linked these tables, allowing us to visualize trends through interactive charts and filters.

### 5. 🎤 Presenting Data to Stakeholders
Finally, what we did was translate these technical findings into a clear narrative for the stakeholders. We focused on presenting the data through storytelling, highlighting key trends and providing data-driven recommendations to help the business improve customer satisfaction and marketing efficiency.
